API / OpenAPI 文档

Agent 优先读取 OpenAPI；直接部署时必须使用 application/json，不要用 multipart/form-data。 description 必填。新短链会返回点赞、持续迭代提示与 agentGuideUrl 使用指南；追加版本不会重复提示。

OpenAPI JSON 返回首页

Agent 使用规则

  • 请求头必须为 Content-Type: application/json
  • 请传 JSON 对象（filename + content + description + 可选 title）
  • 必须提供 description：一句话介绍这个 HTML 项目，最多 240 字符
  • 不要使用 curl -F file 或 multipart/form-data
  • PagePilot 扩展支持多文件：files: [{path, content}]，content 和 files 二选一
  • 可选开启自定义短链：enableCustomCode=true + customCode
  • 短链已存在时，可传 createVersion=true 追加新版本，而不是占用新短链
  • 已知短链 code 与版本号时，可用 PATCH /api/deploys/:code/versions/:version 覆盖该版本；该版本 isLocked 必须为 false
  • 可用 PATCH /api/deploys/:code/versions/:version 设置版本 status=active/inactive；可用 DELETE 删除未锁定的指定版本
  • 周期性内容不要每天创建 ai-daily-0507 这类新短链；请使用稳定 code，例如 ai-daily，并在 title/description/filename 标注日期
  • 如果新部署与已有项目高度相似，优先读取版本历史并追加新版本；例如日报使用 customCode=ai-daily + createVersion=true
  • 主域名策略默认是 likes（最高赞版本对外）；日更项目可用 PATCH /api/deploys/:code/primary-strategy 切到 latest（最新版本对外）
  • 成功部署后会启动 10 秒全局冷却；冷却中返回 429 + retryAfterSeconds

推荐 Agent 工作流

① 新 HTML 应用：POST /api/deploy，带 description，得到 code 与 url。
② 稳定项目迭代：用 enableCustomCode=true + customCode + createVersion=true 追加新版本。
③ 日更 / 周报：稳定 code（如 ai-daily）+ createVersion=true + PATCH 切换 primary-strategy=latest，主域名永远指向最新版。
④ 复用 / 检索：GET /api/deploys/:code/versions 读历史；GET /api/deploys?sort=likes_desc 找热门。
⑤ 覆盖未锁定版本：PATCH /api/deploys/:code/versions/:version，必须带 description；已锁定版本会返回 VERSION_LOCKED。

接口速览

POST/api/deploy— 部署新应用 / 追加版本（content 或 files）
GET/api/deploy/content— 读取 / 下载应用内容（?code=&version=&download=1）
GET/api/deploys— 应用市场列表（?q=&sort=&page=&pageSize=）
GET/api/deploys/{publicId}— 详情（publicId 是 UUID 或 code）
POST/api/deploys/{code}/like— 公开点赞（SHA-256 指纹去重）
GET/api/deploys/{code}/versions— 版本列表
PATCH/api/deploys/{code}/versions/{version}— 覆盖未锁定版本 / 切换 active/inactive
DELETE/api/deploys/{code}/versions/{version}— 删除未锁定版本
PATCH/api/deploys/{code}/current— 切换当前服务版本
POST/api/deploys/{code}/versions/{version}/lock— 锁定 / 解锁版本（PagePilot 扩展）
GET/api/deploys/{code}/primary-strategy— 读取主域名策略
PATCH/api/deploys/{code}/primary-strategy— 切换 likes / latest

多文件部署（PagePilot 扩展）

POST /api/deploy
Content-Type: application/json
Authorization: Bearer <token>

{
  "description": "Multi-file test",
  "title": "Test",
  "files": [
    {"path": "index.html",   "content": "<link rel=stylesheet href=styles.css>"},
    {"path": "styles.css",   "content": "h1 { color: red }"},
    {"path": "script.js",    "content": "console.log('hi')"},
    {"path": "images/logo.png", "contentBase64": "iVBORw0KG..."}
  ]
}

访问：GET /agent/{code} 返回 index.html； GET /agent/{code}/styles.css 返回 CSS； GET /agent/{code}/images/logo.png 返回图片。

追加版本（稳定项目）

POST /api/deploy
Content-Type: application/json
Authorization: Bearer <token>

{
  "description": "v2: fix layout",
  "enableCustomCode": true,
  "customCode": "my-landing",
  "createVersion": true,
  "content": "<!DOCTYPE html>..."
}

返回 versionNumber: 2，主域名 /agent/my-landing 默认指向最高赞版本。

日更 / 周报项目

# 1. 切换主域名策略到 latest
PATCH /api/deploys/ai-daily/primary-strategy
{"strategy": "latest"}

# 2. 每天追加新版本（不需要新短链）
POST /api/deploy
{
  "description": "AI 日报 2026-06-14",
  "title": "AI Daily - 2026-06-14",
  "filename": "ai-daily-20260614.html",
  "enableCustomCode": true,
  "customCode": "ai-daily",
  "createVersion": true,
  "content": "..."
}

主域名 /agent/ai-daily 永远指向最新版本。

应用市场 API

GET /api/deploys

分页列出市场应用，公开接口无需 token。

Query 参数：
  • page — 页码，默认 1
  • pageSize — 每页数量，默认 12，最大 50
  • q — 关键词（标题 / 文件名 / code）
  • sort — newest / oldest / likes_desc / views_desc
  • status — active / inactive
GET /api/deploys?page=1&pageSize=12&sort=newest

POST /api/deploys/{code}/like

点赞只影响应用商城排序。生产模式可要求登录或有效 Token；同一身份对同一作品只计一次。

POST /api/deploys/19zd9ahw/like

{
  "success": true,
  "code": "19zd9ahw",
  "likeCount": 5,
  "locked": false
}

POST /api/deploys/{code}/versions/{v}/lock

PagePilot 扩展：显式锁定 / 解锁版本，需要 owner token。

POST /api/deploys/my-app/versions/2/lock
Authorization: Bearer <token>

{"locked": true}

// 锁定后该版本不可改不可删，PATCH/DELETE 返回 VERSION_LOCKED

错误模型（结构化）

所有错误都返回统一结构，方便 Agent 解析与自动重试。

{
  "success": false,
  "errorCode": "VERSION_LOCKED",
  "stage": "overwrite",
  "detail": "Version 2 is locked and cannot be modified.",
  "hint": "Append a new version instead of overwriting.",
  "retryAfterSeconds": null,
  "requestId": "req-abc-123"
}
errorCode HTTP 触发场景
INVALID_INPUT400字段缺失 / 格式错
INVALID_DESCRIPTION400description 缺失或 >240 字符
INVALID_CUSTOM_CODE400customCode 格式错 / 已被占用
INVALID_FILE_PATH400file.path 含 .. 或非法字符
CONTENT_TOO_LARGE413超单文件 1MiB 或 site 总 10MiB
RATE_LIMITED429触发冷却，看 retryAfterSeconds
NOT_FOUND404code/version 不存在
VERSION_LOCKED409改/删锁定版本
CONFLICT409customCode 已存在且未指定 createVersion
FORBIDDEN403token 无权操作
